CONSOLE FOR EXOTEL

As a number of customers increases, the customer support operations need to streamline. Custom built internal tools can help to achieve that goal.

THE CONTEXT

Exotel was having few tools to help support and account management operations. Some of them were command line tools, those usable only for Engineering teams.

GOAL

The goal of the project was to improve customer experience by following means. Reduce response time of support tickets Provide quality resolutions for customer queries ExoConsole is the tool envisioned to achieve the above goal.

ABOUT THE TEAM

I was working with a product manager who handles internal tools. The engineering team was of two people. I was taking care of entire design process.

UNDERSTANDING THE PROBLEM

To start with we spent time with customer support and account management teams. It resulted in the following insights. There are a lot of to and from communication between customer support and engineering teams related to the tickets. Which results in an increase in response time. The debugging scripts available, are too fragmented. For people without higher technical skills, it is not accessible.

THE APPROACH

The ideas was to come with a tool where people can debug and track issues without any scripts. Over a period of time, the tool will evolve to support all the activities related to the tenants.

FIRST VERSION

The team came up with the first version which supports the following activities. List of issues - ongoing and resolved Tenants affected by those issues Status of support tickets The idea was to give to option to relate the customer support tickets with the ongoing issues. It would be helpful for the support team to respond fast without much help from engineering. The team implemented this version fast, based on paper sketches and whiteboarding. The intent was to use this version for exploring the issues.

SECOND VERSION

This time the goal was to address the issues holistically.

We collected feedback on the first version and did follow up research. Based on that we decided to address the following.

  • How to simplify the debugging of issues
  • How to support adding extra features over time, without breaking navigation pattern.
  • How to organize information for better accessibility
  • How to improve customer communication related to ongoing issues
The Pattern Finder to simplify debugging

To simplify debugging the team came up with something we call Pattern Finder. Where people can input the callsids (unique id for a call). It will display the pattern of the call as given below.

It will also display the possible issues related to that call. For example, say you were facing some issues with a couple of calls yesterday evening. At the same time, Exotel was having some issue with one of the servers. And customer raised a ticket about this today morning. Now using pattern finder customer support person can get all these details via pattern finder. She only needs to input the callsids.

A tool for better customer communication

This version enabled customer communication from the context of an issue. It eliminates the need of going back and forth between tools. Also, it eliminates the possible errors that can happen otherwise.

A customer specific control panel

For each customer, there is a page where all the details for that customer are available. It helps customer support and account management teams to view and resolve customer queries fast.

A better way to share and discuss issues

Each issue has a unique URL. The customer support person can use the copied URL and share it with others to have a discussion.
